4x4 XOR Toggle Light Puzzle

This project implements a classic "Lights Out" style puzzle on an Arduino, where the state of four LEDs is controlled by four buttons using **XOR (toggle) logic**. The challenge was ensuring the puzzle was always mathematically solvable regardless of the initial starting configuration.

Technical Deep Dive: Bitwise XOR for State Management

The core of the puzzle relies on the **Bitwise XOR operator (`^`)**. This operator allows us to efficiently toggle the state of specific LEDs (bits) in a single operation.

The XOR Mechanism

  • The entire puzzle state is held in one 4-bit integer, boardState (range 0-15).
  • Each button is assigned a unique **Pattern Mask** (e.g., MASK_B1).
  • When a button is pressed, the new state is calculated: boardState = boardState ^ mask;
  • If a bit in the mask is 1, the corresponding bit (LED) in the board state flips (0 becomes 1, 1 becomes 0).
  • If a bit in the mask is 0, the corresponding LED is unaffected.

Ensuring Solvability

The key challenge in these puzzles is choosing the right masks so that the resulting matrix of button operations is **invertible**. The following set of masks guarantees that any starting position can be solved (reduced to WIN_STATE = 0).

Button Decimal Mask Binary Mask (B4 B3 B2 B1) Toggles LEDs
B1 14 1110 LEDs 4, 3, & 2
B2 12 1100 LEDs 4 & 3
B3 3 0011 LEDs 2 & 1
B4 7 0111 LEDs 3, 2, & 1

Implementation Highlights

  • Debouncing: Utilized a debouncing function with a 50ms delay to prevent false button presses due to mechanical noise.
  • Random Start State: Used randomSeed(analogRead(A0)) for robust, non-repeating puzzle starts.
  • Binary to Physical Output: The updateLeds() function uses bit shifting (>>) and bitwise AND (&) to translate the single boardState integer into 4 separate digitalWrite() calls.
  • Win Animation: Upon reaching boardState = 0, a flash sequence (binary 1010 / 0101) runs before the puzzle resets.

// Translate the single integer state to physical LED pins
void updateLeds() {
  digitalWrite(ledPin1, (boardState >> 0) & 0x01); // LSB
  digitalWrite(ledPin2, (boardState >> 1) & 0x01); 
  digitalWrite(ledPin3, (boardState >> 2) & 0x01); 
  digitalWrite(ledPin4, (boardState >> 3) & 0x01); // MSB
}
